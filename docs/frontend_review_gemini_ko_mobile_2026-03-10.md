# Gemini Frontend Review: Korean + Mobile

기준일: 2026-03-10

모델:
- `gemini-3.1-pro-preview`

검토 범위:
- `frontend/src/app/App.tsx`
- `frontend/src/components/*`
- `frontend/src/pages/*`
- `frontend/src/styles/app.css`

검토 목표:
- 처음 보는 한국어 사용자 친화성
- 한국어 카피의 자연스러움
- 모바일 사용성
- empty state / 온보딩 / 용어 설명

## Gemini 요약

Gemini가 지적한 핵심은 아래였습니다.

1. 일부 한국어 문구에 줄바꿈 공백 오타가 있어 보인다는 지적
2. 모바일에서 사이드바 메뉴가 홀수 개라서 배치 균형이 애매하다는 지적
3. 첫 사용자 행동 유도는 전반적으로 좋아졌지만, 더 직접적인 CTA를 넣을 여지는 있다는 의견
4. 모바일 테이블 스크롤 가능성을 더 강하게 암시할 수 있다는 의견

## 수용 여부

### 수용

- 모바일 네비게이션 배치 개선
  - `app.css`의 모바일 사이드바 메뉴를 2열에서 3열로 조정
  - 이유: 실제로 5개 메뉴를 2열로 놓으면 마지막 항목이 어색하게 남을 수 있음

### 부분 수용

- 첫 사용자 CTA 강화
  - 이미 `SidebarNav`, `OverviewPage`, `ResearchPage`에 시작 안내와 행동 유도가 들어가 있으므로 추가 구조 변경은 하지 않음
  - 이유: 현재 단계에서 더 늘리면 설명 과잉이 될 수 있음

### 보류

- 줄바꿈 공백 오타 지적
  - 코드 문자열을 다시 확인했을 때 실제 소스에서 재현되지 않았음
  - Gemini CLI 출력 포맷에서 생긴 오탐으로 판단

- 테이블 스크롤 힌트 시각화
  - 현재는 `.table-wrap` + 모바일 `nowrap`로 기본 사용성은 확보됨
  - fade shadow 같은 힌트는 있으면 좋지만 MVP에서는 우선순위가 낮음

- 입력 폼 자동완성/드롭다운
  - 좋은 제안이지만 이번 범위를 넘음

## 반영 후 판단

현재 프론트는 아래 기준에서 충분히 usable 하다고 판단한다.

- 데스크톱에서 첫 사용자가 화면 목적을 이해할 수 있음
- 모바일에서 고정 폭 레이아웃은 제거되었음
- 주요 사용자 노출 문구는 한국어로 정리되었음
- empty state가 다음 행동을 어느 정도 안내함

남은 개선은 대부분 polish 단계이며, 구조 변경이 필요한 blocker는 아니다.
